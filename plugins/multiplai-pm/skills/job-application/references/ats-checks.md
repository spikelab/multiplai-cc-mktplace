# ATS & Screening Optimization

**These checks MUST pass before saving any application. If they fail, the application will likely be auto-rejected.**

## 1. Title & Seniority Alignment

**Problem:** Title mismatch signals wrong fit.

**Checks:**
- [ ] Resume title matches JD title level (Senior PM for Senior PM, not Director/VP)
- [ ] Work history uses REAL titles (background checks verify)
- [ ] Resume body emphasizes BUILDING work over MANAGING work
- [ ] Seniority signals match target level
- [ ] If downleveling from Director → Senior, address in cover letter

**Rule:** Match or go ONE level below JD title, never above.

### Positioning Strategy (Director → Senior PM)

When applying for Senior PM with Director title history:

**Resume:**
- Headline: "Senior Product Manager, AI & Platforms" (matches target)
- Work history: Use real titles — don't lie
- Bullets: Emphasize BUILDING work (platform design, AI strategy, technical decisions) NOT management work
- De-emphasize team size: Move "managed X people" to last bullet or remove
- Lead with building: "Designed architecture" > "Led org of 20"

**Cover Letter Opening (honest AND effective):**
```
After designing agentic AI platforms at [COMPANY], I'm looking for my next
role at a company known for product excellence. Here's why I'm targeting Senior PM
despite Director titles on my resume: I jumped into leadership too fast. I've spent
years coaching entrepreneurs ([REDACTED]), building
teams, and enabling people—I thrive in that work. But I moved into Director roles
at companies that didn't do product management well, and I never got the IC-level
reps of shipping products at a company that does it right. I want to strengthen
those foundations before returning to leadership.
```

## 2. Opening Sentence Complexity

**Checks:**
- [ ] Cover letter opening sentence <25 words
- [ ] Opening contains ONE clear idea
- [ ] No jargon in first sentence
- [ ] Human can explain what you do in <10 seconds after reading opening

**Test:** Read opening aloud. If you take a breath mid-sentence, it's too long.

## 3. Unknown Company Context

**Checks:**
- [ ] Every unknown company gets translated: "[COMPANY] (enterprise messaging platform serving Fortune 500s)"
- [ ] Industry jargon explained on first use
- [ ] Context before achievement

**Rule:** If company isn't in Fortune 500 or top 100 tech startups, add one-line context.

## 4. Polish & Typos

**Checks:**
- [ ] Spell check every word
- [ ] Grammar check
- [ ] Consistent capitalization (AI vs ai, DevOps vs devops)
- [ ] No placeholder text left
- [ ] Numbers formatted consistently

## 5. Visa Sponsorship Strategy

**Checks:**
- [ ] DO NOT mention visa requirement in cover letter or resume
- [ ] DO mention "relocating to [city]" to show commitment
- [ ] DO mention local ties if they exist
- [ ] If asked in form: "Have successfully secured sponsorship at previous roles (Google, [COMPANY]). [REDACTED]."

**Exception:** If JD says "We sponsor visas", mention neutrally.

## 6. Achievement Translation

**Checks:**
- [ ] Every metric is something ACHIEVED, not targeted
- [ ] "Targeting X%" → "Bootstrapped 0→1 product [with specific adoption metric]"
- [ ] If goal isn't achieved, reframe as scope: "Led initiative to..." (no number)
- [ ] Never use future tense

## 7. Career Narrative Coherence

**Checks:**
- [ ] Career arc has clear through-line: "Systems → DevOps → Product → AI Product Leadership"
- [ ] Each transition explained if non-obvious
- [ ] No unexplained gaps >6 months
- [ ] Recent roles show PROGRESSION
- [ ] If applying lower level, frame intentionally

## 8. Domain Translation

**Checks:**
- [ ] Domain-specific terms translated to target domain language
- [ ] Examples use target company's vocabulary
- [ ] Achievements framed in universal terms first
- [ ] If pivoting domains, explicitly bridge

## Pre-Delivery Checklist

### Resume
- [ ] Fits ONE page when rendered (3-4 roles max, 2-3 bullets each)
- [ ] Summary passes 6-second test (specific hook, not generic)
- [ ] Top 5 JD keywords appear naturally
- [ ] Metrics visible without hunting
- [ ] No goals presented as achievements
- [ ] No fabricated facts

### Cover Letter
- [ ] Opens with their need, not my want
- [ ] Under 350 words
- [ ] At least one quantified achievement
- [ ] No filler phrases from kill list
- [ ] Passes out-loud test (would the user say this?)
- [ ] Passes "Would I hire this person?" test
- [ ] No confusion between internal truth and external positioning

### PDF
- [ ] Page 1 is cover letter (NO name header—starts with letter body)
- [ ] Page 2 is resume (complete, no overflow)
- [ ] Resume title fits on one line
- [ ] Contact info in small font (9pt)
- [ ] Inter font renders correctly
- [ ] Dates consistent format throughout

## Enforcement

**Before saving ANY application:**

1. Run EVERY check above
2. If ANY check fails, DO NOT save
3. Report: which checks failed, why, what needs to change
4. Only save after ALL checks pass

**Example failure report:**
```
Cannot save application - failed ATS checks:

❌ Title & Seniority Alignment
   - Resume title: "Associate Director of Product"
   - JD title: "Senior Product Manager"
   - Fix: Change resume title to "Senior Product Manager, AI & Platforms"

❌ Opening Sentence Complexity
   - Current: 52 words, 3 nested concepts
   - Fix: Start with simple hook
```
