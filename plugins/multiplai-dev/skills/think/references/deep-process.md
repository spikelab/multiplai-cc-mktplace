# Deep Mode — Full Decision Process

For high-stakes, complex, or irreversible decisions. Follow phases in order.

## Document

Create a lean tracking document when entering deep mode. Keep it updated before each question.

```markdown

**Status:** Phase N — [phase name] | Paused at Phase N | Complete
**Started:** [date]

## Situation
[Raw facts. What's actually happening before any framing.]

|------|-------|-----------|-----------------|

- **Statement:** I need to [verb] [thing] so that [outcome].
- **Type:** Diagnose | Decide | Design | Plan
- **Stakes:** Low | Medium | High
```

## Phase 1: Understanding

**Goal:** Understand the situation before accepting any problem statement.

1. **Explore first.** Don't accept the stated problem. Ask: What's actually happening? What triggered this?
2. **Generate 2-3 framings.** The stated problem is Frame A. What's Frame B (questions assumptions)? Frame C (zooms out)?
3. **Validate and lock.** Test framings against facts. Select one. Write the problem statement.
4. **Success criteria (HARD STOP).** "What would success look like in a year?" Document 3+ criteria, ranked. Do NOT proceed without these.
5. **Deep discovery.** Clarify values, trade-offs. Apply Hardin's filters: What do the words mean? What are the actual numbers? And then what?
6. **Assumptions table (HARD STOP).** Every assumption gets certainty + impact ratings. Do NOT proceed with an empty table.

**Completion gate:** Situation understood deeply, framings tested, facts investigated, 3+ success criteria, assumptions populated, values documented. All boxes checked before advancing.

---

## Phase 2: Root Cause Analysis

**Only for Diagnose-type problems. Skip otherwise.**

- **5 Whys:** Keep asking why until actionable root cause. At each level: what incentives are at play?
- **IS / IS NOT:** What has the problem vs. similar things that don't? Where/when does it occur vs. not?
- Before finalizing: systemic fragility or one-time event?

---

## Phase 3: Divergent Ideation

**No evaluation. Generate first, judge later.**

Contribute options — don't just critique. This is where most sessions fail.

1. Propose 1-2 options using Strategist lens
2. Get reaction
3. Propose more based on response
4. Continue until 10+ options exist (you contribute at least 5)
5. Track count: "That's 4 options. Here's another angle..."

**Hard rules:**
- Minimum 10 options before proceeding
- Must span different strategic directions (do more, do less, do differently, abandon, delay)
- Always include "Do Nothing / Delay" as explicit option
- No criticism yet

---

## Phase 4: Stress Testing

**Entry check:** 10+ options documented? You contributed 5+? If not, return to Phase 3.

- **Premortem:** "6 months later, this failed badly. Why?" List 3-5 failure modes.
- **Consider-the-opposite:** Strongest argument against leading option.
- **Reference class:** What typically happens in similar situations?
- **Scenarios:** Test top options against optimistic, base, and adverse cases.
- **Handling uncertainty:** Remove unlimited-downside options. Prefer options preserving future choices.

---

## Phase 5: Evaluation

1. Score each option against success criteria from Phase 1
2. Rank with explicit reasoning
3. Head check (makes sense logically?) + Heart check (feels right?)
4. Type 2 (reversible): satisfice. Type 1 (irreversible): maximize.

If user leans toward a weaker option, challenge it. Let them override but make them defend it.

---

## Phase 6: Quality Gate

| Element | Rating (0-100%) |
|---------|-----------------|
| Helpful Frame | |
| Clear Values | |
| Creative Alternatives | |
| Useful Information | |
| Sound Reasoning | |
| Commitment to Follow Through | |

Identify weakest link. If too low for these stakes, return to the relevant phase.

---

## Phase 7: Conclusion

- Document chosen path with reasoning
- Identify kill conditions (what would change our mind)
- List concrete next actions with owners
- Write 3-5 bullet brief summary

**Phase Skip Protocol:** When user wants to skip: (1) Challenge — "This phase exists for [reason]." (2) Probe — "Are you avoiding something hard?" (3) If they insist, document and proceed.

**Session Management:** Mark "Paused at Phase N" with date if stopping mid-process. The document should contain everything needed to resume cold.
